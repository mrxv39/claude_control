# UX Audit: Claudio Control

**Date**: 2026-04-16
**Scenario**: "Quiero monitorizar mis sesiones Claude abiertas, ver cuál necesita atención, enfocar una sesión, y revisar la salud de mis proyectos en el panel del orquestador"
**Persona**: Desarrollador con múltiples proyectos, usa Claude Code a diario, primera vez usando Claudio Control. Sabe qué es una sesión pero no conoce la UI.
**Method**: Code-based review (no browser — Electron desktop app)
**Viewport**: Bar 300-600×48px + Panel 400px

## Summary

La barra principal es compacta y funcional: chips con color de estado, badges de git y contexto comunican mucho en poco espacio. Sin embargo, la app tiene **problemas de discoverability** — funciones importantes (Ctrl+click para tile, click en service badge) son completamente invisibles para un usuario nuevo. El panel del orquestador empaqueta mucha información pero carece de feedback tras acciones del usuario, y la configuración mezcla conceptos avanzados sin explicación. La app es potente para power users pero hostil para primeros usos.

## Flow Score

| Metric | Value |
|--------|-------|
| Task completed? | Partially — monitoring works, but advanced features hidden |
| Click count | 1 click per focus, 2 clicks for panel tab |
| Decision points (had to stop and think) | 4+ (qué es el dot? qué significa IDLE vs WAITING? cómo tile?) |
| Dead ends (wrong path, had to backtrack) | 1 (cerrar panel con ✕ de la barra oculta todo) |
| Would come back? | Yes — once you learn it |
| Could teach someone in 2 minutes? | No — tile, service badges, pacing need explanation |
| One thing to make it twice as easy | Tooltips contextuales en primer uso |

## Findings

### Critical (blocks user task)

*Ninguno* — la app no bloquea tareas. Todo es accesible eventualmente.

### High (causes confusion or friction)

- **H1: Ctrl+click para tile es invisible**: No hay ninguna indicación visual de que Ctrl+click existe. El tile es una de las features más útiles pero un usuario nuevo jamás la descubrirá.
  - *Where*: Chip click handler (`index.html:133-140`)
  - *Fix*: Añadir un tooltip sutil al primer chip hover ("Ctrl+click para tile") o un hint en la barra cuando hay 2+ sesiones.

- **H2: WAITING vs IDLE — misma señal visual, distinto significado**: Ambos usan dot rojo y fondo rojo. WAITING = Claude terminó y espera input (urgente). IDLE = no hay Claude (no urgente). Un usuario nuevo no distingue cuál necesita atención.
  - *Where*: `styles.css:53-65` — `.chip.waiting` y `.chip.idle` usan el mismo `--red`
  - *Fix*: IDLE debería tener un color diferente (gris/dim) o al menos opacity más baja. Actualmente `.idle .dot` tiene `opacity:.6` pero el chip background es casi idéntico.

- **H3: Toast notification no es interactiva**: El toast "X terminó — esperando tu input" aparece 5s y desaparece. No puedes hacer click para enfocar esa sesión. Pierdes la notificación si no la viste.
  - *Where*: `notifications.js:16-41` — `toast.setIgnoreMouseEvents(true)`
  - *Fix*: Hacer el toast clickable (remove `setIgnoreMouseEvents`), con click → `focus-wt` de esa sesión.

- **H4: El botón ✕ de la barra oculta toda la app, no solo el panel**: Si tienes el panel abierto y pulsas ✕, no cierra el panel — oculta la barra entera al tray. Un usuario nuevo que quiere cerrar el panel usará ✕ instintivamente.
  - *Where*: `index.html:15` — `onclick="ipcRenderer.invoke('hide-bar')"`
  - *Fix*: Si el panel está abierto, ✕ primero cierra el panel. Segundo click oculta la barra. O hacer el ✕ del panel más prominente.

- **H5: Chime bloquea el main process ~0.5s**: `playChime()` usa `execFileSync` (síncrono). Durante ese medio segundo, la barra no responde a clicks. Con múltiples sesiones terminando rápido, puede congelar varios segundos.
  - *Where*: `notifications.js:117-119` — `execFileSync('powershell.exe', ...)`
  - *Fix*: Usar `execFile` (async) o Web Audio API en el renderer.

### Medium (suboptimal but workable)

- **M1: No hay feedback al guardar configuración**: Click en "Guardar" en la sección Configuración del tab Cola simplemente recarga el tab. No hay toast, no hay confirmación visual. El usuario no sabe si se guardó.
  - *Where*: `index.html:521-533` — `saveConfigBtn` handler
  - *Fix*: Flash el botón en verde, o mostrar "Guardado" temporal.

- **M2: Stats tab — labels en español/inglés mezclados**: "Ciclo 5h en vivo" (español), pero "Stats" (inglés) como nombre del tab. "Costes" (español) pero "skill" (inglés). Inconsistencia de idioma.
  - *Where*: `index.html:661-770` y panel tabs
  - *Fix*: Elegir un idioma y ser consistente. Dado que los labels principales están en español ("Salud", "Cola", "Log"), todo debería ser español.

- **M3: Empty state de la Cola no es accionable**: Dice "Se encolarán automáticamente fuera de horario" pero no ofrece un botón directo para encolar algo manual ahora. El botón "+ Tarea" existe arriba, pero el texto del empty state no lo menciona.
  - *Where*: `index.html:456-457`
  - *Fix*: Añadir un CTA en el empty state: "Encolar tarea manualmente" que abre el mismo dialog.

- **M4: El select de prioridad no tiene feedback visual inmediato**: Cambiar la prioridad de un proyecto recarga todo el tab Health. Si tienes muchos proyectos, pierdes tu posición de scroll.
  - *Where*: `index.html:359-362`
  - *Fix*: Actualizar solo el card afectado, o mantener scroll position tras reload.

- **M5: Context % badge y context bar redundantes**: Cada chip Claude muestra el porcentaje como badge (`45%`) Y como barra de progreso debajo. Doble información, ocupa espacio.
  - *Where*: `index.html:97-111`
  - *Fix*: Quitar el badge numérico y dejar solo la barra de contexto (con tooltip para el número exacto), o viceversa.

- **M6: "No sessions" aparece sin contexto**: Si no hay sesiones Claude, muestra "No sessions" en inglés, sin sugerir qué hacer.
  - *Where*: `index.html:39`
  - *Fix*: "Sin sesiones activas. Abre Claude Code en Windows Terminal." en español.

- **M7: Scheduler debug info expuesta al usuario**: El tab Stats muestra "offHours: true · idle: false · busy: false · pacing: burst" — esto es debug info, no es accionable para el usuario.
  - *Where*: `index.html:688-689`
  - *Fix*: Mover a un collapse "Debug" o eliminar del UI principal.

- **M8: Service badges sin indicación de que son clickables**: Los badges `⚙ tauri dev` parecen etiquetas informativas. Nada indica que son clickables para enfocar esa ventana.
  - *Where*: `styles.css:79-80` — `.svc` no tiene `cursor:pointer` visual cue
  - *Fix*: Añadir underline on hover, o un icono de "open/focus" al hover.

### Low (polish)

- **L1: `esc()` function name is ambiguous**: `esc()` es la función de escape HTML. Funciona pero el nombre es críptico en debugging.

- **L2: Gear icon (⚙) como texto Unicode**: En algunas fuentes/resoluciones puede renderizar distinto. Considerar un SVG inline.

- **L3: Panel tabs no tienen indicador de contenido**: No se sabe si la Cola tiene tareas pendientes, o si hay proyectos con score bajo en Salud, sin abrir cada tab.
  - *Fix*: Añadir badge numérico en los tabs (ej: "Cola (3)", "Salud ⚠").

- **L4: Update chip uses mixed language**: "Update ${version}" en inglés, pero el tooltip dice "Click para descargar la nueva version" en español (sin tilde).
  - *Where*: `index.html:219-220`

- **L5: Toast message has typo**: "terminó" should have accent → "terminó" (actually correct in code, but "termino" without accent in `notifications.js:53`).
  - *Where*: `notifications.js:53` — `termino` sin tilde

- **L6: Overlay font size (16px) is large**: El overlay de título en la barra de WT usa 16px bold, que puede ser demasiado prominente comparado con el texto nativo de WT.

- **L7: Stats chart has no Y-axis label**: El gráfico de barras de "Uso por ciclo" no indica que el eje Y es porcentaje.

- **L8: Horizontal scrollbar on chips container hidden**: `.chips::-webkit-scrollbar { display:none }` — si hay muchas sesiones, no hay indicación visual de que hay más chips a la derecha.
  - *Fix*: Fade gradient en el borde derecho cuando hay overflow.

## Emotional Friction

- **Momento 1 — "¿Qué significan estos colores?"**: Al ver la barra por primera vez, veo dots verdes y rojos. Verde = ocupado (intuitivo al revés — normalmente verde = bien, rojo = mal). Pero aquí verde = Claude trabajando (bien) y rojo = esperando (necesita atención). La inversión es correcta para el contexto pero no obvia sin explicación.

- **Momento 2 — "¿Se guardó mi config?"**: Cambié el horario del scheduler y pulsé Guardar. El panel recargó. ¿Se guardó? No hubo ninguna confirmación. Siento incertidumbre.

- **Momento 3 — "¿Cómo pongo dos ventanas lado a lado?"**: Quiero comparar dos sesiones. No veo ningún botón de tile. Resulta que es Ctrl+click (descubierto por tooltip casual), pero nada en la UI lo sugiere.

- **Momento 4 — "¿Para qué es este indicador morado?"**: El idle indicator en la barra muestra `5h:14%→40% BURST CAP`. Sin contexto previo, esto es incomprensible. ¿Qué es "burst"? ¿Qué es "CAP"?

## What Works Well

- **Chip design es excelente**: Compacto, informativo, color-coded. Branch + dirty count + context % en un espacio mínimo.
- **Auto-tile automático**: Redistribuir ventanas WT automáticamente cuando abres/cierras sesiones es brillante y ahorran mucho tiempo.
- **Overlays en WT**: Ver el proyecto + estado directamente sobre cada ventana WT sin cambiar de contexto es muy útil.
- **Agrupado por proyecto**: Sesiones Claude + servicios del mismo proyecto en un solo chip elimina clutter.
- **Health scoring**: El tab Salud da una visión rápida de qué proyectos necesitan atención con score visual.
- **Stats con SVG inline**: El gráfico de uso por ciclo está bien hecho sin dependencias externas.
- **Pacing system**: El sistema de pacing es sofisticado y bien diseñado para maximizar uso del rate limit.
- **Context degradation bar**: La barra de contexto con colores progresivos (verde→amarillo→rojo) comunica urgencia intuitivamente.
- **Debounce en notificaciones**: 3 polls consecutivos para evitar falsos positivos es un buen detalle.

## Recommendations (priority order)

1. **Hacer el toast clickable** (H3) — máximo impacto con mínimo esfuerzo. Cambiar `setIgnoreMouseEvents(true)` a false, añadir click handler.

2. **Diferenciar visualmente IDLE vs WAITING** (H2) — IDLE en gris/dim, WAITING mantiene rojo. Comunica urgencia correctamente.

3. **Añadir hint de Ctrl+click para tile** (H1) — tooltip en primer hover, o un badge "Ctrl+click" sutil cuando hay 2+ sesiones.

4. **Feedback al guardar config** (M1) — flash de "Guardado" en el botón o toast interno.

5. **Hacer chime async** (H5) — `execFile` en lugar de `execFileSync`. Evita bloqueos del main process.

6. **Hacer ✕ context-aware** (H4) — si panel abierto, cerrar panel; si panel cerrado, ocultar barra.

7. **Badges en panel tabs** (L3) — "Cola (3)" para indicar tareas pendientes sin abrir el tab.

8. **Consistencia de idioma** (M2) — todo en español, incluyendo tabs y labels.

9. **Overflow indicator en chips** (L8) — gradient fade cuando hay más chips ocultos a la derecha.

10. **Limpiar debug info del UI** (M7) — mover scheduler debug a collapsible o eliminar.
